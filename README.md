# TOAP - Review Artifact

This artifact accompanies the TOAP paper submission. It contains the
implementation, configuration, and a fully reproducible pipeline for the
**CIFAR-10 / ResNet-18 (BadNets)** experiments, plus implementation code and
configuration templates for the remaining experiments.

> Scope summary: fully reproducible CIFAR-10\ResNet18 and
> The ImageNet-100 and multimodal (BLIP-2 / LLaVA) experiments are provided as
> implementation code, configuration templates.

## Directory layout

```
artifacts/
  README.md                  - this file
  MANIFEST.txt               - file tree, sizes, SHA256 checksums, versions
  env/                       - environment specs (conda + pip)
  source/                    - source code (CIFAR-10 core + baselines + other-exp code)
  configs/
    cifar10/                 - exact CIFAR-10 configs + frozen protocol trigger banks
    other_experiments/       - ImageNet-100 / watermark config templates
  scripts/                   - setup + run + verify + clean helper scripts
  results/
    cifar10/                 - reference CIFAR-10 metric tables (paper values)
    other_experiments/       - existing ImageNet-100 / multimodal result tables
  logs/cifar10/              - reference fixed-bank metric logs
  checkpoints/cifar10/       - (no binaries shipped; see README there)
  docs/                      - scope, data, anonymization notes
```

## Quick start (CIFAR-10)

1. Create the environment:
   ```bash
   bash scripts/setup_env.sh
   conda activate toap_fnl_env        # if conda is available
   ```
2. Run the lightweight smoke test (downloads CIFAR-10, trains 1 epoch, evaluates):
   ```bash
   bash scripts/run_cifar10_smoke.sh
   ```
3. Run the full CIFAR-10 reproduction (downloads CIFAR-10, trains the backdoored
   model, fixed-bank evaluation; see the script for downstream TOAP stages):
   ```bash
   bash scripts/run_cifar10_full.sh
   ```
4. Verify produced metrics against the reference values:
   ```bash
   bash scripts/verify_cifar10_results.sh
   ```
5. Remove generated outputs (keeps source/configs/results/logs/checkpoints):
   ```bash
   bash scripts/clean_outputs.sh
   ```

## How CIFAR-10 source is organized

The CIFAR-10 entrypoint is `source/main.py` (training of clean and backdoored
ResNet-18 + fixed-bank W-ASR evaluation). It imports sibling modules
(`data.py`, `model.py`, `train.py`, `protocol_banks.py`), so it is run as
`python source/main.py ...` from the artifact root. The TOAP topological
pipeline (`extract_activation_cifar10.py` -> `build_correlation_matrix_*` ->
`compute_persistence_from_corr_*` -> `compute_tss_*` -> `rl_edge_pruning_*` ->
`auto_vaccination_revised.py`) and the adaptive evaluation
(`source/scripts/eval_adaptive_final_patch.py`) operate on the checkpoints and
artifacts produced by `main.py`. Each stage exposes a `--help`.

## Reproducibility notes / honest limitations

- Only the CIFAR-10 experiments are fully reproducible from this artifact.
- No trained checkpoints (`*.pth`) are shipped; they are regenerated by running
  the pipeline (see `checkpoints/cifar10/README.md`).
- The included CIFAR-10 configs (`configs/cifar10/`) reference the original
  paper run directories/checkpoint paths; when reproducing from scratch you
  must repoint them at your freshly produced run directory.
- Results may vary slightly due to non-deterministic GPU operations.
- The reference metric files under `results/` and `logs/` are the values used
  in the paper and are provided unchanged (only absolute paths were sanitized).
